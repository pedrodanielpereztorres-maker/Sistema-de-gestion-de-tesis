### Stack tecnológico
- Framework: **Reflex 0.9.1** (Python reactivo, componentes = funciones Python que retornan rx.Component)
- Base de datos: **PostgreSQL** — conexión via psycopg2 con pool SQLAlchemy
- ORM: **NO usa Reflex ORM** — usa SQL puro con cursores psycopg2 directos

- Reportes: **fpdf2** para PDF, **csv** nativo para exportaciones
- Deploy: LAN local, acceso multiusuario simultáneo

### TAREA 8 🟠 IMPORTANTE — Query SQL dentro de `@rx.var` agota el pool de conexiones

**Archivo:** `estado/estado_estudiante.py` — propiedad `estudiantes_por_carrera`

**El problema exacto:**
```python
@rx.var
def estudiantes_por_carrera(self) -> list[dict]:
    conn = obtener_conexion()           # Abre conexión de BD
    with conn.cursor() as cursor:
        cursor.execute("SELECT ...")    # Ejecuta query SQL
    conn.close()
```
En Reflex, los `@rx.var` son propiedades computadas que se recalculan
CADA VEZ que cualquier variable del estado cambia — esto puede ocurrir
10-20 veces por segundo durante una interacción normal. Cada llamada
abre una conexión y ejecuta una query, lo que agota el pool de conexiones
(`max_overflow=20`) y bloquea hilos del servidor.

**Qué debes hacer:**
1. Agregar una variable de estado: `estadisticas_carrera: list[dict] = []`
2. Crear método `async def _cargar_estadisticas_carrera(self)` que haga la query
   y llene `self.estadisticas_carrera`
3. Llamar `await self._cargar_estadisticas_carrera()` al final de `cargar_estudiantes()`
4. Convertir `estudiantes_por_carrera` en un `@rx.var` puro que NO toque BD:
   ```python
   @rx.var
   def estudiantes_por_carrera(self) -> list[dict]:
       conteo: dict[str, int] = {}
       for e in self.lista_estudiantes:
           carrera = e.get("carrera", "Sin Carrera")
           conteo[carrera] = conteo.get(carrera, 0) + 1
       total = len(self.lista_estudiantes) or 1
       return [
           {"nombre": k, "cantidad": v, "progreso": min((v / total) * 100, 100)}
           for k, v in sorted(conteo.items(), key=lambda x: x[1], reverse=True)
       ]
   ```

Muéstrame los cambios en `estado_estudiante.py`.
